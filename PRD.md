> **버전**: v1.0  
> **작성일**: 2026-04-09  
> **상태**: 확정 (개발 착수 가능)  
> **대상 독자**: 개발팀, 디자이너, 경영진

## 1. 서비스 개요

### 1.1 서비스 정의

**FeedbackFlow**는 조직의 비전/미션/핵심가치/조직목표를 사전에 세팅해두면, 팀원과 리더가 **개략적인 내용만 입력**하더라도 조직 목표와 정렬된 **구조화된 셀프리뷰 및 리더 피드백을 자동 생성**하는 AI 어시스턴트 웹 서비스이다.

### 1.2 서비스가 아닌 것

- HR/성과관리 시스템 자체를 대체하지 않는다.
- 인사 평가 점수를 산출하거나 등급을 매기지 않는다.
- 결과물은 **초안**이며, 사용자가 반드시 검토/수정 후 활용한다.

### 1.3 핵심 가치 제안

| 사용자 | 기존 문제 | FeedbackFlow 해결 |
|--------|----------|-------------------|
| 팀원 | 셀프리뷰가 단순 업무 나열 수준. 목표/핵심가치와 연결 못함 | 키워드만 입력하면 KPT 구조로 목표 연계 리뷰 자동 생성 |
| 리더 | 팀원별로 톤/깊이가 다른 피드백 작성에 1인당 30분~1시간 소요 | 셀프리뷰 + 간단 코멘트 입력 → 상황별 톤에 맞는 피드백 초안 즉시 생성 |
| HR | 조직 전체 피드백 품질 편차. 핵심가치와 괴리된 피드백 | 모든 피드백이 핵심가치/조직목표와 자동 연계되어 일관성 확보 |

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)

```
팀원: "이번 분기에 한 일을 나열해야 하는데, 뭘 어떻게 써야 할지 모르겠다"
       → 결과: 업무 목록 나열 수준의 셀프리뷰

리더: "팀원 5명 피드백을 써야 하는데, 비슷한 말만 반복하게 된다"
       → 결과: 추상적이고 일관성 없는 피드백

HR:   "피드백이 핵심가치랑 전혀 연결이 안 되어 있다"
       → 결과: 성과 리뷰가 조직 방향성과 괴리
```

### 2.2 문제 구조화

| # | 문제 | 근본 원인 | 영향 |
|---|------|----------|------|
| P1 | 셀프리뷰가 업무 나열에 머무름 | 목표 대비 성과를 구조화하는 프레임워크 부재 | 리뷰의 전략적 가치 없음 |
| P2 | 조직 목표/핵심가치와 단절 | 리뷰 작성 시점에 목표/가치를 참조하지 않음 | 개인-조직 정렬도 파악 불가 |
| P3 | 리더 피드백 품질 편차 | 리더마다 피드백 역량/시간 차이 | 팀원 성장 기회 불균등 |
| P4 | 피드백에 실행 가능한 액션 부재 | "잘했다/부족하다" 수준에서 끝남 | 피드백 후 행동 변화 없음 |

### 2.3 목표 상태 (To-Be)

```
팀원: 3~5줄 키워드 입력 → AI가 KPT 구조 + 목표 연계 + 핵심가치 매핑까지 완성
리더: 핵심 코멘트 1~2줄 + 톤 선택 → AI가 구조화된 피드백 + 액션 아이템까지 생성
HR:   모든 리뷰/피드백이 조직 목표와 핵심가치에 자동 연계
```

## 3. 타겟 사용자

### 3.1 타겟 조직

- **규모**: 5~50인 스타트업 및 중소기업
- **성과관리 성숙도**: 레몬베이스/FLEX/노션/구글시트 등으로 성과관리를 시도하고 있으나, 리뷰/피드백 "콘텐츠" 품질에 고민이 있는 조직
- **업종 제한**: 없음 (핵심가치/목표를 직접 입력하므로 업종 무관)

### 3.2 페르소나 정의

#### 페르소나 A — 팀원 (김주니어, 28세, 마케터)

```
역할:       마케팅팀 주니어
성과관리:   분기별 KPT 셀프리뷰 작성
Pain:       "뭘 어떻게 써야 좋은 리뷰인지 모르겠다. 
             항상 비슷한 말만 반복하게 된다."
Want:       짧게 써도 잘 정리된 리뷰가 나왔으면 좋겠다
행동패턴:   키워드 중심으로 빠르게 입력하고 싶어함
성공 기준:  입력 시간 10분 이내, 리더가 "잘 썼다"고 느끼는 결과물
```

#### 페르소나 B — 리더 (박팀장, 36세, 팀장)

```
역할:       8명의 팀원을 관리하는 팀장
성과관리:   분기별 1on1 + 연말 종합 피드백
Pain:       "8명 피드백을 쓰려면 하루가 꼬박 걸린다.
             상황마다 톤을 다르게 써야 하는 게 가장 어렵다."
Want:       핵심만 입력하면 상황에 맞는 피드백이 자동으로 나왔으면
행동패턴:   셀프리뷰를 읽고 바로 핵심 코멘트를 남기는 식으로 진행
성공 기준:  팀원 1명당 피드백 작성 시간 5분 이내
```

#### 페르소나 C — HR 관리자 (이매니저, 33세, People팀)

```
역할:       조직 전체 성과관리 프로세스 설계/운영
Pain:       "팀마다 피드백 품질이 천차만별. 
             핵심가치 연계? 아무도 안 한다."
Want:       핵심가치/조직목표가 자동으로 반영된 피드백 체계
행동패턴:   온보딩에서 조직 정보를 세팅하고, 결과물을 모니터링
성공 기준:  조직 전체 피드백에서 핵심가치 언급률 80% 이상
```

## 4. 기능 상세 정의

### F1. 온보딩 — 조직 컨텍스트 설정

**목적**: AI가 리뷰/피드백 생성 시 참조할 조직 컨텍스트를 세팅한다.

#### 입력 항목

| 필드 | 필수 여부 | 입력 방식 | 예시 |
|------|----------|----------|------|
| 회사명 | 필수 | 텍스트 | "리스닝마인드" |
| 미션 | 필수 | 텍스트 (500자 이내) | "We guide the path to marketing truth" |
| 비전 | 선택 | 텍스트 (500자 이내) | — |
| 핵심가치 | 필수 (최소 1개) | 반복 입력 폼 | 아래 상세 |
| 조직 목표 | 선택 | 반복 입력 폼 | 아래 상세 |
| 피드백 가이드라인 | 선택 | 텍스트 (1000자 이내) | "SBI 모델을 기반으로 피드백한다" |

#### 핵심가치 입력 구조

```json
{
  "name": "의도 탐구에서 시작하자",
  "nameEn": "Intent-Centric Exploration",
  "description": "고객의 진짜 욕구를 파헤쳐라. 그것이 모든 가치 창출의 출발점이다.",
  "behaviors": [
    "고객/시장의 의도를 먼저 파악하고 업무에 반영한다",
    "표면적 요구가 아닌 본질적 니즈를 탐구한다"
  ]
}
```

- 최대 10개까지 등록 가능
- 각 핵심가치당 행동지표(behaviors)는 최대 5개

#### 조직 목표 입력 구조

```json
{
  "title": "Q1 마케팅팀 목표",
  "description": "인텐트 기반 캠페인 전환율 15% 달성",
  "team": "마케팅팀",
  "year": 2026,
  "quarter": 1
}
```

#### 비즈니스 규칙

- BR-01: 핵심가치가 1개 이상 등록되어야 AI 기능 사용 가능
- BR-02: 온보딩 완료 후에도 언제든 수정 가능
- BR-03: 조직 목표는 연도/분기별로 관리

### F2. 목표 설정 어시스턴트 (Phase 2)

**목적**: 팀원이 작성한 성과/성장 목표 초안을 SMART 기준으로 고도화한다.

#### 입력/출력

```
[입력]
팀원 작성: "이번 분기에 마케팅 성과를 높이고 싶다"

[AI 처리]
1. 조직 목표와 연관성 분석
2. SMART(Specific, Measurable, Achievable, Relevant, Time-bound) 기준 적용
3. 핵심가치 연계 행동지표 추천

[출력]
성과목표 (안):
"Q1 인텐트 기반 캠페인 3건 기획·실행하여 전환율 12% → 15% 개선 (3월 말 기준)"
- 연계 조직목표: 마케팅팀 Q1 목표 (인텐트 기반 캠페인 전환율 15% 달성)
- 연계 핵심가치: 의도 탐구에서 시작하자, 인텐트 데이터로 마케팅 통합을 리딩하자
- 측정 기준: 캠페인 수 3건 + 전환율 15% 달성 여부
```

#### 비즈니스 규칙

- BR-04: AI 제안은 항상 "제안"이며, 사용자가 수정/확정한다
- BR-05: 목표는 성과목표/성장목표 두 유형으로 구분하여 관리
- BR-06: 확정된 목표는 이후 셀프리뷰/피드백 생성 시 컨텍스트로 자동 주입

### F3. 셀프리뷰 작성 AI (MVP 핵심)

**목적**: 팀원이 개략적으로 입력한 내용을 KPT 프레임워크로 구조화하고, 합의된 목표 및 핵심가치와 자동 매핑한다.

#### 화면 구성 (입력)

```
┌─────────────────────────────────────────────────┐
│  셀프리뷰 작성                    [2026 Q1 ▼]    │
├─────────────────────────────────────────────────┤
│                                                  │
│  참조 목표 (자동 로드)                             │
│  ┌─────────────────────────────────────────┐     │
│  │ [성과] 인텐트 캠페인 전환율 15% 달성      │     │
│  │ [성장] GA4 데이터 분석 역량 확보           │     │
│  └─────────────────────────────────────────┘     │
│                                                  │
│  이번 분기에 한 일, 느낀 점을 자유롭게 적어주세요   │
│  ┌─────────────────────────────────────────┐     │
│  │ 신규 캠페인 3건 런칭함                     │     │
│  │ 전환율 14% 정도 나온 것 같음               │     │
│  │ 데이터 분석은 좀 부족했음                   │     │
│  │ GA4 강의 들으려고 했는데 시간이 없었음       │     │
│  │ 다음엔 꼭 GA4 마스터하고 싶음              │     │
│  └─────────────────────────────────────────┘     │
│                                                  │
│              [AI로 리뷰 생성하기]                  │
│                                                  │
└─────────────────────────────────────────────────┘
```

#### AI 처리 로직

```
입력:
  - 사용자 자유 텍스트
  - 해당 분기 합의된 성과/성장 목표 (자동 주입)
  - 회사 핵심가치 + 행동지표 (자동 주입)
  - 조직 목표 (자동 주입)

처리:
  1. 자유 텍스트에서 성과/활동/감정/계획 요소 추출
  2. KPT(Keep/Problem/Try) 프레임워크로 분류
  3. 각 항목을 합의된 목표에 매핑
  4. 관련 핵심가치 및 행동지표 연결
  5. 구체성이 부족한 부분에 보강 가이드 추가

출력:
  - KPT 구조화된 셀프리뷰 텍스트
```

#### 화면 구성 (출력)

```
┌─────────────────────────────────────────────────┐
│  AI 생성 셀프리뷰                  [복사] [수정]  │
├─────────────────────────────────────────────────┤
│                                                  │
│  Keep (유지할 점)                                 │
│  ──────────────────────────────────              │

│       14% 달성 (달성률 93%)                       │
│     - 핵심가치 연계: '인텐트 데이터로 마케팅         │
│       통합을 리딩하자' — 인텐트 데이터 기반으로      │
│       캠페인을 설계하고 실행까지 완수함              │

│     - 핵심가치 연계: '지속 성장과 공유의 문화를      │

│     💡 보강 가이드: 구체적으로 어떤 상황에서         │
│        데이터 분석이 필요했는지 사례를 추가하면      │
│        리뷰의 설득력이 높아집니다                   │
│                                                  │
│  Try (시도할 점)                                  │
│  ──────────────────────────────────              │
│  1. Q2 중 GA4 기초~중급 과정 이수                  │
│     - 성장목표 연계: 데이터 분석 역량 확보          │
│     - 목표 기한: 2026년 6월 말                    │
│  2. 캠페인 전환율 15% → 18% 상향 도전             │
│     - 성과목표 연계: 전환율 목표 초과 달성 시도      │

│                                                  │
│  ─────────────────────────────────               │
│  목표 달성 요약                                    │
│  [성과] 캠페인 전환율 15% → 14% 달성 (93%) 🟡     │
│  [성장] GA4 역량 확보 → 미착수 (0%) 🔴            │
│                                                  │
│         [이대로 확정] [다시 생성] [직접 수정]        │
│                                                  │
└─────────────────────────────────────────────────┘
```

#### 비즈니스 규칙

- BR-07: AI는 사용자가 입력하지 않은 성과를 창작하지 않는다 (환각 방지)
- BR-08: "보강 가이드"는 선택적 제안이며, 최종 결과물에 포함되지 않는다
- BR-09: 목표 달성도 수치는 사용자 입력 기반이며, AI가 임의로 산정하지 않는다
- BR-10: 생성된 리뷰는 [복사] 버튼으로 클립보드에 복사 가능 (사내 HR 툴에 붙여넣기용)
- BR-11: [다시 생성] 시 동일 입력으로 다른 표현의 결과를 생성 (temperature 조절)

### F4. 리더 피드백 생성기 (MVP 핵심)

**목적**: 팀원의 셀프리뷰와 리더의 개략적 코멘트를 기반으로, 상황에 맞는 톤과 구조를 갖춘 피드백을 자동 생성한다.

### F5. 연말 종합 피드백 (Phase 2)

**목적**: 4개 분기의 셀프리뷰 + 리더 피드백을 종합하여 연간 성과 평가 초안을 생성한다.

#### 입력

- 해당 팀원의 Q1~Q4 셀프리뷰 (자동 수집)
- 해당 팀원의 Q1~Q4 리더 피드백 (자동 수집)
- 연초 합의 목표 (성과 + 성장)
- 리더의 종합 코멘트 (자유 텍스트)

#### 출력 구조

```
1. 연간 성과 요약
   - 주요 성과 하이라이트 (분기별)
   - 성과목표 달성률 종합

2. 성장 궤적
   - 연초 → 연말 역량 변화 서술
   - 성장목표 달성 여부

3. 핵심가치 실천도
   - 핵심가치별 실천 사례 종합
   - 가장 잘 실천한 가치 / 보강 필요한 가치

4. 종합 피드백
   - 올해의 강점
   - 내년 중점 개선 영역

5. 차기연도 목표 제안
   - 성과목표 방향 제안
   - 성장목표 방향 제안
```

#### 비즈니스 규칙

- BR-18: 4개 분기 중 최소 2개 분기 데이터가 있어야 생성 가능
- BR-19: 누락 분기가 있는 경우 명시적으로 표기 ("Q3 데이터 없음")

### 7.2 Prisma 스키마 (핵심)

```prisma
enum UserRole {
  ADMIN
  LEADER
  MEMBER
}

enum GoalType {
  PERFORMANCE
  GROWTH
}

enum GoalStatus {
  DRAFT
  AGREED
}

enum ReviewStatus {
  DRAFT
  GENERATED
  CONFIRMED
}

enum FeedbackPurpose {
  GROWTH
  IMPROVE
  WARNING
  PRAISE
}

enum FeedbackTone {
  FIRM
  COACHING
  NEUTRAL
  ENCOURAGING
}

model Company {
  id          String      @id @default(cuid())
  name        String
  mission     String      @db.Text
  vision      String?     @db.Text
  guideline   String?     @db.Text
  coreValues  CoreValue[]
  orgGoals    OrgGoal[]
  users       User[]
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
}

model CoreValue {
  id          String   @id @default(cuid())
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id])
  name        String
  nameEn      String?
  description String   @db.Text
  behaviors   String[] // 행동지표 배열
  order       Int      @default(0)
}

model OrgGoal {
  id          String   @id @default(cuid())
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id])
  title       String
  description String   @db.Text
  team        String?
  year        Int
  quarter     Int?
}

model User {
  id              String       @id @default(cuid())
  companyId       String
  company         Company      @relation(fields: [companyId], references: [id])
  email           String       @unique
  name            String
  role            UserRole     @default(MEMBER)
  team            String?
  goals           Goal[]
  selfReviews     SelfReview[]
  givenFeedbacks  Feedback[]   @relation("FeedbackAuthor")
  receivedFeedbacks Feedback[] @relation("FeedbackTarget")
  createdAt       DateTime     @default(now())
}

model Goal {
  id          String     @id @default(cuid())
  userId      String
  user        User       @relation(fields: [userId], references: [id])
  type        GoalType
  title       String
  description String     @db.Text
  year        Int
  quarter     Int?
  status      GoalStatus @default(DRAFT)
}

model SelfReview {
  id            String       @id @default(cuid())
  userId        String
  user          User         @relation(fields: [userId], references: [id])
  year          Int
  quarter       Int
  rawInput      String       @db.Text   // 사용자 원본 입력
  aiOutput      String?      @db.Text   // AI 생성 결과 (JSON)
  finalOutput   String?      @db.Text   // 사용자 확정 결과
  goalSnapshot  Json?                    // 생성 시점 목표 스냅샷
  status        ReviewStatus @default(DRAFT)
  feedbacks     Feedback[]
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  @@unique([userId, year, quarter])
}

model Feedback {
  id            String            @id @default(cuid())
  reviewId      String?
  review        SelfReview?       @relation(fields: [reviewId], references: [id])
  authorId      String
  author        User              @relation("FeedbackAuthor", fields: [authorId], references: [id])
  targetId      String
  target        User              @relation("FeedbackTarget", fields: [targetId], references: [id])
  year          Int
  quarter       Int
  purpose       FeedbackPurpose[]
  tone          FeedbackTone
  leaderComment String?           @db.Text
  aiOutput      String?           @db.Text  // AI 생성 결과 (JSON)
  finalOutput   String?           @db.Text  // 리더 확정 결과
  status        ReviewStatus      @default(DRAFT)
  createdAt     DateTime          @default(now())
  updatedAt     DateTime          @updatedAt
}
```

AI 생성 결과는 Server-Sent Events(SSE)로 스트리밍하여 사용자가 실시간으로 결과가 타이핑되는 것을 볼 수 있도록 한다.

```typescript
// /api/reviews/[id]/generate (Route Handler)
export async function POST(req: Request, { params }: { params: { id: string } }) {
  const review = await prisma.selfReview.findUnique({
    where: { id: params.id },
    include: { user: { include: { company: { include: { coreValues: true, orgGoals: true } } } } }
  });

  const goals = await prisma.goal.findMany({
    where: { userId: review.userId, year: review.year, quarter: review.quarter }
  });

  const prompt = buildReviewPrompt({
    company: review.user.company,
    goals,
    rawInput: review.rawInput,
  });

  const stream = await anthropic.messages.stream({
    model: 'claude-sonnet-4-6-20250416',
    max_tokens: 2000,
    system: prompt.system,
    messages: [{ role: 'user', content: prompt.user }],
  });

  return new Response(stream.toReadableStream(), {
    headers: { 'Content-Type': 'text/event-stream' },
  });
}
```
